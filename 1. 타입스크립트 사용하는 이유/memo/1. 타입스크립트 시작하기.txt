#1 타입스크립트를 쓰는 이유 

1) 왜 타입스크립트를 쓰는 건가?
 - 우리가 사용하는 크롬, 엣지, 사파리 등의 브라우저에서는
   타입스크립트를 이해하지 못한다. 

 - 결국 브라우저가 이해할 수 있는 언어인 JS로 컴파일을
   해줘야한다. 
   
 이런 과정이 번거로울 거 같은데 왜 타입스크립트를 
 사용하는 게 좋은 지 알아보자!


 - 너무 관대한 자바스크립트
   타입 지정을 맘대로 하거나 값을 넣지 
   않아도 문제가 전혀 없다. 
    → 원하는 결과값이 나오지 않을 수 있다.   
    → 자바스크립트 엔진은 아무런 힌트를 
      얻지 못한다.   
   
 - 타입스크립트를 쓰는 이유 
  ◎ 자바스크립트는 실행되는 시점(= 런타임)에 타입이
    결정되고 이때 오류를 발견한다. 

  ◎ 반대로 Java, TypeScript 같은 정적언어에서는 
    컴파일 시점에 타입이 결정되고 컴파일 시점에 오류를 
    발견한다.  
 
  ◎ 다른 사람이 만든 함수를 사용할 때, 몇 개의 인수를 
    어떤 타입으로 사용해야할 지 찾아보지 않아도 어떤
    타입이 와야하는지 알려준다. 

  ◎ 에러가 있는 코드는 런타임전(저장하기전) 오류를 
    보여주므로 개발자의 실수 방지할 수 있다.   
     


2) 설정없이 타입스크립트 사용하기 
  → https://www.TypeScriptlang.org/play
  → vscode에서 사용하려면 패캠 온라인 강의 정리한 거 참조!
  → 기존 코드를 작성해서 결과를 확인해보자

  function showItems(arr) {
      arr.forEach(item => { //Parameter 'arr' implicitly has an 'any' type.
          console.log(item) //Parameter 'item' implicitly has an 'any' type.
      });
  }

  showItems([1, 10, 20]);
  showItems(1, 2, 3) //Expected 1 arguments, but got 3.

  - any 타입
   → any 타입은 어떤 타입이든 사용해도 되는 타입으로 
     정말 어떤 타입인지 모를 때 사용한다.
   → 가급적 사용하지 않는 게 좋다.
